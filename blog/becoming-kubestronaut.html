Becoming a CNCF Kubestronaut:
Passing All 5 Kubernetes Certifications

Roman Rabodzei · May 2026 · ~10 min read
Kubernetes CNCF Certifications CKA CKAD CKS

In early 2026 I earned the CNCF Kubestronaut designation — awarded to engineers who pass all five Kubernetes certifications: KCNA, KCSA, CKA, CKAD, and CKS. This post covers my honest experience with each exam: what I used to study, what surprised me, and what actually matters in practice.

KCNA KCSA CKA CKAD CKS Kubestronaut

Why all five?

I had been working with AKS in production for a few years before deciding to get formally certified. I started with CKA to validate the fundamentals, but once I had it I kept going. The KCNA and KCSA cover cloud-native concepts and security theory — useful for understanding the ecosystem beyond just Kubernetes itself. CKAD sharpened my application-development perspective on the platform. And CKS, the hardest one, pushed my security knowledge into territory I genuinely use at work: runtime security, supply chain hardening, and cluster-level threat mitigation.

Once CKA, CKAD, and CKS were done, KCNA and KCSA felt like a natural close to the loop. The Kubestronaut badge is a nice bonus, but the real value was filling in the gaps.

The exams, one by one

KCNA — Kubernetes and Cloud Native Associate

Multiple-choice, no terminal. Covers Kubernetes fundamentals, container concepts, cloud-native architecture, and CNCF project landscape. If you already work with Kubernetes day-to-day this will feel straightforward. The trickiest parts are the broader ecosystem questions about CNCF projects you may not use directly — Fluentd, Jaeger, Linkerd concepts.

Prep tip: Read through the CNCF landscape and understand what each major project category does (observability, service mesh, CI/CD, security). Don't memorise — understand the purpose.

KCSA — Kubernetes and Cloud Native Security Associate

Also multiple-choice. Security-focused theory: cluster hardening, network policies, supply chain security, runtime security, and compliance frameworks. A good bridge between KCNA and CKS. Expect questions on the 4Cs model (Cloud, Cluster, Container, Code), Pod Security Standards, and RBAC concepts.

CKA — Certified Kubernetes Administrator

Performance-based, 2 hours, 17 questions in a live terminal. This is where things get hands-on. Core topics: cluster bootstrapping with kubeadm, etcd backup and restore, troubleshooting broken nodes and control-plane components, RBAC, storage (PV/PVC), networking (Services, Ingress, NetworkPolicies), and upgrade procedures.

  • Bookmark the Kubernetes docs — you can use them during the exam. Know where things are.
  • Practice etcdctl snapshot save/restore until it's automatic.
  • Time management matters more than depth. Move on and return to hard questions.
  • Use kubectl explain and --dry-run=client -o yaml constantly.

CKAD — Certified Kubernetes Application Developer

Same format as CKA but developer-focused. Multi-container Pods, init containers, ConfigMaps and Secrets, resource limits, probes, Helm basics, Jobs and CronJobs. If you've done CKA first, CKAD covers a lot of overlapping ground but from a different angle — less cluster admin, more application lifecycle.

Tip: The CKAD has more questions per hour than CKA. Speed is everything. Practice generating resource manifests from scratch with kubectl run and kubectl create flags rather than writing YAML by hand.

CKS — Certified Kubernetes Security Specialist

The hardest one. Requires an active CKA to sit. Topics include: API server hardening, audit logging, Falco runtime threat detection, AppArmor and seccomp profiles, image scanning with Trivy, OPA/Gatekeeper policies, network policies, mTLS with Istio concepts, and supply chain security. The exam environment can have multiple clusters and you must be very careful about context switching.

  • Know Falco rules syntax — you'll likely need to modify one.
  • Understand the difference between PodSecurityAdmission enforce/warn/audit modes.
  • Practice AppArmor profile loading and referencing in Pod specs.
  • Trivy: trivy image --severity HIGH,CRITICAL <image> — know the flags.
  • NetworkPolicies: isolate namespaces, deny-all defaults, selective allow rules.

Resources I actually used

  • Killer.sh — Two free simulator sessions are included with exam registration. They're harder than the real exam. Do them multiple times.
  • KillerCoda — Free browser-based scenarios. Great for CKS-specific topics like Falco, AppArmor, and Trivy.
  • Official Kubernetes docs — The canonical reference. Learn to navigate it quickly.
  • Certified Kubernetes courses on Udemy — Mumshad Mannambeth's CKA/CKAD/CKS courses. Solid fundamentals coverage.

Practical takeaways

Earning these certifications solidified concepts I thought I already knew — and exposed a few gaps I didn't realise were there. CKS in particular changed how I approach cluster security reviews at work: I'm now much more deliberate about audit logging, runtime detection, and admission controller coverage.

If you're working with Kubernetes in production and considering this path: start with CKA, follow with CKS while your CKA knowledge is fresh, then round out with CKAD. Add KCNA and KCSA whenever convenient — they're lower effort if you already have the practitioner certs.

The Kubestronaut designation is a nice milestone but the process itself is the real value. Good luck — and keep your kubectl shortcuts sharp.
